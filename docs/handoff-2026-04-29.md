## Root cause (confirmed)

The Sprite Line Buffer (SLB) inside `vgc_sprites.sv` — 320×7 bits — was being mapped by yosys's `memory_libmap` to **distributed RAM** (LUT4_D / TRELLIS_DPR16X4 cells), NOT to `DP16KD` block RAM. The cascade-write logic for cell-banked dual-port distributed RAM **silently drops writes**. So `SPR_CLEAR`'s 320 sequential per-scanline writes never actually clear the buffer — sprite data accumulates forever.

Three symptoms collapse to one cause:
- Bar persists across cold_starts (BRAM contents survive rst, only FFs reset).
- Bar is full screen height (no per-scanline clears, sprite writes persist).
- Bar accumulates at every previous sprite X (each new write adds a stuck slot).
- Bar color (black/white/gray) varies per POR — uninitialized LUT-RAM cells.
